`examples` 目录下的内容是开发者专门为用户准备的“教学 Demo”，它们通常比 `src` 里的脚本更简单、更具针对性。

以下是对图片中 `examples` 文件夹内各个子目录的作用解析，以及你的**第一步**该怎么走：

---

## 1. `examples` 文件夹内容解析

| 文件夹名 | 作用与学习价值 |
| :--- | :--- |
| **`tutorial`** | **最推荐的起点！** 包含从 0 到 1 的基础教程，比如如何加载数据集、如何进行简单的推理。 |
| **`training`** | 专门展示如何训练模型。相比根目录那个复杂的 `train.py`，这里的脚本通常更精简，适合初学者理解训练循环。 |
| **`dataset`** | 教学如何自定义数据集。对于你的“舒适抓取”研究，如果你以后要录制自己的数据，这里是必看的参考。 |
| **`port_datasets`** | 教你如何将其他格式的机器人数据集（如官方的 RLDS 格式）转换成 LeRobot 格式。 |
| **`so100_...` / `phone_...`** | 这些是针对**特定硬件**的示例。比如 `so100` 是一款低成本开源机械臂，`phone` 是教你如何用手机遥控机器人。 |
| **`backward_compatibility`** | 处理旧版本代码兼容性的示例（一般研究可以先不用看）。 |
| **`rtc`** | 即 Real-Time Communication（实时通信），涉及远程控制或低延迟传输的示例。 |

---

## 2. 第一步：运行 `tutorial`

既然已经 clone 了代码并了解了依赖安装，建议从 **`examples/tutorial`** 开始。这里面通常包含：
1. **`1_load_lerobot_dataset.py`**: 如何调取 Hugging Face 上的机器人数据集。
2. **`2_visualize_dataset.py`**: 如何把机械臂的动作画出来。

### 操作步骤：
在终端执行（确保你在 `lerobot` 根目录下）：

```bash
# 运行第一个教程：加载数据
python examples/tutorial/1_load_lerobot_dataset.py
```


---

## 3. 为什么先跑这里而不是直接跑 `src`？

* **代码量少**：`tutorial` 里的脚本通常只有几十行，一眼就能看清逻辑。
* **不依赖复杂配置**：`src` 里的脚本（如 `train.py`）使用 `Hydra` 配置系统，需要处理大量的 `.yaml` 文件，初学者很容易配置晕。而 `examples` 里的代码通常是硬编码参数，直接运行即可。
* **即时反馈**：你能快速看到数据加载是否成功，这能帮你确认你的环境（Python 依赖）是否真的装好了。
